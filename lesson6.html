Introduction

Assume we have an array of integers. Let’s say it’s a big one. Size 1000. Let’s say we want to fill it up with 1s. Are you excited to write 1000 lines of code to fill it up!

If you said yes, you are weird. If you’re normal, you will love this lesson about loops. Loops are ways we can tell the computer to do something over and over again. This way, we don’t have to write 1000 lines of code. We tell the computer to stop by telling it to check if a boolean is false. If it’s false, we must stop the loop. In a sense, this aspect of the loop is similar to an if statement that tells the computer whether or not to run a command. That might’ve been a bit confusing. I know. Let me try to help you out as much as possible. We will be introducing two types of loops today. The for-loop and the while-loop. I will provide as much explanation and examples as possible to help you guys learn this concept! It’s a hard one, but we will get through it!

For-Loops

Let’s look at an example of a for-loop. Try to guess what you think it does!

int[] arr=new int[1000];
for(int i=0;i<1000;i++){
arr[i]=0;
}
What we are doing here is first creating an integer i, that we start at 0. We do this within the () after the “for”. Then, we check if i is less than arr.length, which is put after a “;” after declaring int i=0. If this is true, we can run the loop. Here, we simply set arr[i] to equal 0. “i” is 0 right now, so arr[0]=0. Then, our code does i++, which we put after the “;” after the boolean statement. This is the same thing as i=i+1. All we are doing here is incrementing i so we can set the next element to equal to 0.

Eventually, i will equal 999. We set arr[999]=0, then add 1 to i. Now i is equal to 1000. We check i<1000. However, this is now false. Thus, the loop ends because of this.

To summarize, we have four parts to a for-loop. The first part declares a new variable, such as int i=0;. This happens before the loop even starts, and only happens once. The second part happens at the start each time we run the loop, where we check a boolean such as i<100. If this boolean is true, we run the loop. If it isn’t, the loop ends. The third part of the for-loop is what we do after the loop runs each time. For example, we do i++ because we need to change i for the loop to go through the whole array. The last part is the stuff we do in the loop! In the example, we set arr[i]=0;

While Loops

The while loop is another very popular kind of loop. I believe it’s a lot simpler than for loop. However, you will see more for-loops than while loops, which is why I introduced it first. However, the while loop is still extremely important and useful!

Like a for-loop or an if statement, the while loop depends on a boolean. In fact, it depends solely on a boolean. While the boolean is true, the loop will run. Once the boolean isn’t true anymore, the loop will stop running. Check out the following example.

boolean a=true;
while(a){
System.out.println(1);
}
 Here, the while loop checks if the boolean within the parentheses is true. In this case, since set a to true, it will always evaluate to true. Thus, the loop will always run.

What’s wrong with this while loop? Can you spot it?

Well, the loop never ends since the boolean will always evaluate to true. Thus, this will run an infinite amount of times. This is not okay. It will crash.

Let’s give an example of a while loop that does work.
int i=0;
while(i<10){
i++;
}
 Here, “i” starts at 0. We then check to see if i<10, which is true. Since it’s true, we run our code inside the loop, which adds 1 to i. It repeats this process until i is equal to 10, where then i<10 is false. Now the loop ends and the computer moves on to the next lines of code.
