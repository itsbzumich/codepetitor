Lesson 4

Conditional Statements and Booleans

 

Introduction

What if we wanted the computer to not always do the same thing? For example, let’s say we want to print “Yes” if a number variable is above 10, and “No” if a number variable is less than 10. Well, we could write two different codes and run them. But that’s not efficient. In this case, we must use Conditional Statements. This allows the code to do different things based on the conditions. In a conditional statement, we check to see if a boolean, a true or false value, is true or not. If it is, we will then execute the code inside the brackets. If not, we’ll do something else. It’s okay if it’s confusing right now. By the end of this lesson, you’ll understand a lot better!

Here’s an example before we dive into more detail.
if(true){
System.out.println(“Hello World);
}
 This code checks if the boolean value inside the “if()” is true or false. If it’s true, and since it is, it will print out Hello World!

Booleans

In order to understand how conditional statements work, we need a greater understanding of booleans. This is because in order to trigger an if statement, we need to pass in a boolean value. Thus, we’ll demonstrate here how to generate such boolean values.

The most basic idea is directly using a boolean variable. For example:

boolean x=true;
 Then, we can say if(x) as our conditional statement. if(x) is true of course.

int x=0;
x==10;

If we simply did x=10, we would be setting x to ten. Using "==", we will generate true if x is equal to ten. It will generate false if x is not equal to ten. Thus, if we wrote:
if(x==10){
x=1;
}
 This code checks to see if x=10. Since it doesn't, it will set x equal to one. Be sure you know the difference between == and =.

What if we wanted to do something if x isn’t equal to ten? Then, we would use !=, which means “not equal to”.
if(x!=10){
System.out.println("Hello World");
}
 This will only print Hello World if x is not equal to ten.

Our final case is to use < or> to generate a boolean. For example, let’s say we have an integer grade that stores a grade. We can write if(grade>90) for example. If grade>90, it will be true. We use ">" and "<" just like we would in math. We can also use>= to symbolize greater than or equal to. <= would mean less than or equal to.

To conclude this section, we will introduce a rather advanced topic. We can also combine booleans. Let’s say we have two booleans a and b. Let’s say we only want to do something if both a and b are true. We can write if (a&&b). && is the symbol for and.

On the other hand, let’s say we want to do something if only one of a or b is true. In that case, we would write a||b. This means a or b. || means or. a||b is true if only one of a or b is true. Both don’t need to be true.

One more important topic. What if we want the opposite of a boolean? What we want to do if(a isn't true?). Then, we use !a. The ! mark makes whatever's next to it the opposite. So now, if a is true, !a is false. If a is false, !a is true.

To summarize, we can directly call a boolean, use == to generate a boolean, use >/=/<= or use !=to generate a boolean. We can combine multiple booleans with && or ||. ! means "not" . It makes a true value false and a false value true.

If Statements

We've already mentioned if statements earlier. They are the ones that use "if". We will dive a little more in depth here.

if(boolean){
System.out.println(“true”);
}
If the boolean inside the parentheses is true, then we will run the statement inside the brackets, which is to print true. If it’s false, then we skip over the if statement and move on.

Else Statements

Let’s assume that we have two cases. We want our program to do one thing if x is equal to 10, but do another thing if x is not equal to ten. Let’s try using two if statements for this. if(x!=10){
x=10;
}
if(x==10){
System.out.println(“yes”);
}
Can you spot the mistake? In this code, if x is not equal to 10, we switch it to equal to 10. Now x is equal to 10. Then, when we check if the next if statement is true, we realize it’s true, so we print “yes” as well. Therefore, when x is initially not 10, it’s doing both of the things inside the brackets, which we don’t want.

If we want one or the other but not both, we can use the else statement. Here’s an example of it.
if(x!=10){
x=10;
}
else{
System.out.println(“yes”);
}
 Now, if x!=10 initially, it will only run x=10. It won’t print yes as well. The else statement essentially states that if(x!=10), it will run x=10 and ONLY x=10. To that end, if x=10, it will print “yes”. It will first check to see if x!=10. Since x=10, it will run the else command instead.

Else If Statement

Let’s say we want to test many conditions, but only want one to run. In this case, we will use else if statements along with if statements.

if(x==1){
System.out.println(“one”);
}
else if(x==2){
System.out.println(“two”);
}
else{
System.out.println(“invalid”);
}

Essentially, else if statements function as mini else statements with a restriction. If we add an else statement at the end, it will run the command within the else statement if and none of the else-if statements are true. Another way to think about it is that it will check all the statements until it finds one that's true. When it does, it runs that command and ends without checking the others.

Here’s a quick summary before we start our practice problems.
• If is the initial statement that does something if the boolean is true.
• Else if statements after an if statement say that assuming the if is false, do the else if statement command. Does the first else if that's true.
• Else is that if none of the if or else if booleans are true, then do the else statement command.

 

Functions/New Problem Format

Before we start the next set of practice. I would like to introduce a new format of a question.
void q1(boolean a){
Insert your code here.
}
 This has a different header and looks kind of weird. It's called a function. We write it outside of the main code. Let's discuss how it works.
Basically, we have an input and an output. Whats inside the (), in this case, boolean a, is something that we don't know. It's out input, and can be anything. a=true or a=false. We will have to write our code without knowing the value of this boolean. q1 is the name of the question and don’t worry about what void means. The reason I’m going to start using functions is that I can specify random variables that you must use within your code. In this case, I’m giving you a boolean called a that I will define myself, but you won’t know what it is. For example, in the example code up bellow, let's say we had to print out the value of a. We would simply just write System.out.println(a); In other words, assume whatever is inside the parenthesis is already declared but has an unknown value. Feel free to reach out if you need help!
Example Question: If boolean a is true, print “I love you”. If it isn’t, print out “NO”.
Answer:
void q1(boolean a){
if(a){
System.out.println(“I love you”);
}
else{
System.out.println(“NO”);
}
}
